Сайт инжиниринговой компании и внутренний сервис по созданию заказов интегрированный c Б24 и 1С, каталог товаров

Технологический стек :

PYTHON
DJANGO
DRF -для REST API
CELERY
PostgreSQL
Проектирование БД
Настройка сервера VPS/Linux,nginx
WAL-G бекапирование

Интеграции :

ЦБ рф - курс валют
Производственный календарь для высчитывания срока действия счетов
dadata - для лк клиентов на сайте и заполнении информации юрлиц
smsru - логин с одноразовым паролем
битрикс24 - получение информации о юрлицах клиентов. Синхронизация заказов
1С:УНФ - отправка данных по заказам. Получение остатков склада
Настройка сервера VPS/Linux,nginx
API поставщиков получение товаров и данных
Заказчик : инжиниринговая компания с крупным департаментом продаж
Задача : Создание внутреннего сервиса объединенного каталог товаров из каталогов поставщиков. Создание полнофункционального сервиса сборки заказов и генерации документов менеджером интегрированного с б24 1с унф и сайтом. Разработка клиентского сайта. Автоматизации бизнес-процессов Заказчика
Данный проект оказался крупным, на него уже ушло более 1500 часов работы. Также я столкнулась со сложными, нетиповыми задачами в разработке. На данном проекте я выступала в качестве бекенд разработчика и архитектор всего проекта. Также мной настраивалась серверная часть, прорабатывались и ставились задачи для фронтенд-разработчика в зависимотси от потребностей бекенда.
Разрабатываемая система в первую очередь агрегирует данные о товарах - получает и обновляет данные от 6 крупных поставщиков и форматирует их к единому формату. У части партнеров было свое API из которого мы берем данные по товарам-ценам-наличию и т.п. Для другой части были разработаны индивидуальные скрипты разбора файлов прайсов XLSX и CSV. Проведена интеграция товаров из 1С в нашу систему для синхронизации и создания новых товарных позиции. ТАкже для системы ОКТ(Объединенного товарного каталога) Введена система скидок поставщиков в зависимости от товарных групп. Для управление каталогом и его параметрами разработа админка на основе django admin. Так-же разработана система уровней доступа разным сотрудникам к разным частям функционала всей системы.
Cледующей задачей была разработка сервиса сборки заказов для менеджеров компании и интеграции сервиса в систему Битрикс24. Сервис сборки заказа открывается во вкладке заказа в Б24 в iFrame и забирает api-запросами к битрикс необходимую для создания сделки и документов информацию. Если заполнена не вся необходимая информация выдается предупреждение что надо заполнить. Дальше менеджер набирает корзину товаров и выставляет необходимые параметры скидок/дат поставок и т.п. Та же в корзине добавлена возможность самостоятельно создания нового товара которого нет в ОКТ. После сохранения заказа создается первичная документация из переданных ранее параметров и записывает в необходимые поля в битрикс24. Также собираются данные и передаются в 1С:УНФ. Для сайта также создана система личного кабинета пользователей и создании заказов. В этом случае система сохраняет в себе все необходимую первичную для сделки информацию и создает в Б24 новую сделку которую в дальнейшем обрабатывают менеджеры по схеме вышке. Из 1С:УНФ Система получает информацию по движению товара на складе и произведенным отгрузкам и оплатам по заказу. Также транслирует эту информацию в Битрикс24 и лк клиента. Также в системе предусмотрены уведомления для менеджеров битрикс о изменении закупочных цен на товары в заказах и повышении курса валют.
Благодаря проработанной системе логирования изменений на всех этапах по заказу и проводимых действий удается отслеживать проблемы возникающие у сотрудников время производимых действий
